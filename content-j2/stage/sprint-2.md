We zagen nog wel een warning voorbij komen dat ik beter `globalThis` inplaats van `window` kan gebruiken, dus kreeg ik een uitleg over waarom het beter is om [globalThis](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/globalThis) te gebruiken. Dat komt omdat een `window` en Node.js niet goed samenwerken. Het probleem met `globalThis` is dat de global van een [iframe](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/iframe) (inner frame, dus een mini websiteje in je grote website) anders is dan de global van je outerframe (je hoofd pagina). Maar inmiddels gebruikt bijna niemand dat meer dus kan ik volgens Robbert veilig `globalThis` gebruiken. Toen ik dat wilde aanpassen gaf vsCode een error aan dar `globalThis` undefined is. Dit kwam hoogstwaarschijnlijk omdat de `es2020` het nog niet zou ondersteunen. Dus deze mocht ik updaten naar `es2022` en `es2023`. Maar om dit te laten werken moet je ook in de Eslint config aangeven dat dit mag. Dus heb ik `es2020: true,` aan de `env: {}` toegevoegd onder de andere configuraties en daarna werkte het wel. <br>

In de middag kwam Robbert bij mij om te vragen wat ik aan het uitspoken was, ik vertelde dat ik bezig ben met de style dictionary in mijn stage repo aan de praat te krijgen maar dat ik een error heb die ik aan het uitzoeken ben. Hij kwam even met mij meekijken. 
- In de `config.json` stond nog het standaard `source": ["tokens/**/*.json"]` maar omdat mijn bestand `fdnd.tokens.json` heet moeten de sterretjes voor het woord tokens staan. De sterretjes vervangen namenlijk elk karakter. De enkele * zoekt alleen tussen de / in het path waar de ster tussen staat. En de dubbele ** kijkt over het gehele path en vervangt de sterretjes met karakters. Dus met `source": ["/**/*.tokens.json"]` pakt hij vanuit alle mapjes de `fdnd(want de sterretjes worden vervangen door fdnd).tokens.json`. Nu zoekt hij overal in de repo naar dit bestand dus om het concreet te maken geef je aan dat je vanuit het `src` mapje wilt zoeken `source": ["src/**/*.tokens.json"]`.
- De tokens in het json bestand die ik van figma heb gekregen zitten een laag te diep. Hierdoor kunnen ze niet gelezen worden. Met `pnpm add @tokens-studio/sd-transforms` heb ik de Tokens Studio preprocessor plugin geinstalleerd. Dit zorgt ervoor dat de bovenste laag wordt overgeslagen. In het `style-dictionary.mjs` bestand zet je een `import { register } from "@tokens-studio/sd-transforms";` neer en `register(StyleDictionary, {excludeParentKeys: true,});`. Zorg dat dit voor/boven het stukje code staat waarin je de tokens wilt gaan maken/lezen. Anders heb je mosterd na de maaltijd. Zonder dit stapje staat er `"$value": "{common.basis.text.line-height.md}"` inplaats van `"$value": "{basis.text.line-height.md}"`.
- Door `"log": {"verbosity": "verbose"},` aan de `config.json` toe te voegen krijg je een uitgebreidere error melding in de terminal.
- Bij de `pnpm run build` ging hij daadwerkelijk zonder errors door en heeft hij een mapje `build` aangemaakt. Hierin staan nu de mapjes `android` en `scss` die ik niet nodig heb en ik kan veranderen naar `css`
- Uit een voorbeeld van de [themes repo](https://github.com/nl-design-system/themes/blob/dedfc700dc0f1807024960565371e67a3fdbac62/style-dictionary-config.js#L137) kon ik spieken hoe ik een dist mapje maak waar ik css variablen in aanmaak. Door de code die je in [deze commit](https://github.com/frameless/stage-anouk/commit/be600b491fa4d2f4ee861255d83011911835da0f#diff-587cb980af76fdc7e52369fd0b9d926dff266976b6f8ac631e358fecc49ff8cf) ziet toe te voegen en `pnpm run build` in de terminal te zeggen is het mapje en de bestanden aangemaakt.
