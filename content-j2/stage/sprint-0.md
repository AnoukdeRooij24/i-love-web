## Donderdag 5-3-2026
In de standup heb ik vertelt wat ik gister heb gedaan en dat ik tegen een probleem aanloop wat ik ga uitzoeken vandaag. Ik kan geen tokens toevoegen in tokens studio in figma. Dit probleem zat hem in de json naam. Hierna heb ik de fdnd kleuren, border radius en fonts toegevoegd aan de tokens studio op brand en common niveau.
om 11 uur heb ik meegedaan aan de Developer Open Hour #87 in het nl-design-system-developers kanaal. Hier heb ik mezelf voorgesteld en daarna ging het over de design token small paragraph, of je een kleinere size dan 16px door mag blijvn doorvoeren. Ook was er een mini quizje over waar je op zou zoeken bij verschillende componenten zodat dit in de beschrijving bij het component gezet kan worden zodat het sneller gevonden kan worden via een zoek functionaliteit. En daar is een lux, eslint, node error besproken omdat eslint geupdate moet worden van 8 naar 9. En we kregen een preview van de nieuwe documentatie van het design system nl handboek op hun website.
Na de lunch heb ik van Bart een rondleiding gekregen door de [theme wizzard website](https://theme-wizard.nl-design-system-community.nl/style-guide). Hij heeft uitgelegd hoe ze het hebben gemaakt en hoe het werkt. Je vult een url in, een scraper (enige serverside js op de website) haalt alle css van de website af en parsed dit naar design tokens. Deze tokens kan je vervolgens plaatsen op de plekken waar je dit wilt ter vervanging van het start thema. [Wij hebben dit met de fdnd site gedaan](https://theme-wizard.nl-design-system-community.nl/). Hij heeft ook zijn eigen website [Wallace](https://www.projectwallace.com/get-css) laten zien waar de scraper van het design system op gebaseerd is. Hierop kan je alle css informatie van een website zien waarvan je de url invult.
Daarna hebben we samen gekeken naar hoe ik de huisstijl van fdnd in het format van de nl design system kan krijgen. Dit was nog best een hele puzzel omdat de fdnd site meer action kleuren heeft dan dat er in het nl design system beschikbaar zijn. Hierom hebben we gekeken of we de paarse button als inverse versie van de gele button kunnen krijgen. En hebben we naar de rest van de kleuren gekeken, nu is het aan mij de taak om deze in figma tokens te zetten.
Voor robberts idee van een website maken waar je een nl design sysyem json bestand in kan geven en een url van een website had ik geen idee hoe ik dat zou kunnen doen. Door `npx @nl-design-system-community/theme-wizard-server` in de terminal te runnen krijg je een soort mini lokaal CSS scrapertje ([bron](https://www.npmjs.com/package/@nl-design-system-community/theme-wizard-server)). Deze haalt dan de info van een url die je doorgeeft en rekent dit om naar design tokens. Vervolgens kan je aan de hand van [colorjs.io](https://colorjs.io/) uitrekenen (of uit laten rekeken) hoeveel verschil er tussen de kleuren zit met 'maxDeltaE'. Als je die kleuren naast elkaar legt dan kan je simpel laten zien welke kleur er beter gebruikt kan worden in het geval van dat de kleur nog niet in het design system staat. Maar dit is voor een later moment om uit te voeren. Ook heeft hij [pnpm install](https://pnpm.io/installation) en [nvm install](https://github.com/nvm-sh/nvm) gedaan.
Ik heb grotendeels alle color tokens in figma tokens studio kunnen zetten.
